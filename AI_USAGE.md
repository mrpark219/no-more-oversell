# AI_USAGE.md

- 이 문서는 프로젝트 진행 과정에서 AI 도구를 어떻게 활용했는지 정리한다.
- AI는 기술적 판단을 대체하는 도구가 아니라, 요구사항 구체화, 선택지 비교, 오류 사전 발견을 위한 보조 도구로 사용했다.
- 최종 설계 방향, 구현 범위, trade-off, 테스트 실행 여부는 직접 검토 후 결정했다.

# 1. 활용 원칙

1. AI 제안을 그대로 사용하지 않고 현재 코드베이스와 요구사항에 맞는지 검토했다.
2. 코드와 문서 내용이 다르면 실제 구현 코드를 기준으로 수정했다.
3. 재고, 주문, 결제 정합성과 같은 핵심 로직은 DB 제약, 트랜잭션, 테스트 결과를 기준으로 판단했다.

# 2. 활용 범위

## 2.1 요구사항 구체화

- 모호한 요구사항을 질문 단위로 나누고 구현 범위를 정리하는 데 사용했다.
- deep-interview 방식으로 다음 내용을 구체화했다.
    - 목표
    - 구현 범위
    - 제외 범위
    - 완료 기준
    - 운영 환경에서 남는 한계

## 2.2 설계 판단 보조

- 주요 설계 선택지와 trade-off를 비교하는 데 사용했다.
- AI는 선택지 탐색에 활용했고, 최종 결정은 구현 복잡도와 정합성 검증 가능성을 기준으로 내렸다.

## 2.3 구현 보조

- service, domain, repository, exception 처리 코드 작성 시 보조 도구로 활용했다.
- 반복적인 구조 작성에 주로 사용했다.
- 제안된 구조가 현재 코드베이스와 맞지 않으면 수정하거나 제외했다.

## 2.4 테스트 코드 작성 보조

- 단위 테스트, service 테스트, API 테스트, 동시성 테스트 작성에 활용했다.

## 2.5 오류 사전 발견

- 코드와 문서 간 불일치나 빠진 예외 흐름을 찾는 데 활용했다.

## 2.6 문서화 보조

- README, DECISIONS, AI_USAGE 문서의 구조와 표현을 정리하는 데 사용했다.
- 구현되지 않은 기능을 구현된 것처럼 설명하지 않는 원칙을 유지했다.

# 3. 최종 검증 방식

1. AI가 제안하거나 생성한 내용은 다음 기준으로 검증했다.
    - 실제 endpoint와 service 흐름 확인
    - DB migration의 unique/check 제약 확인
    - 단위 테스트 및 통합 테스트 실행 결과 확인
    - 부하 테스트 결과 확인
2. 최종 책임은 구현자에게 있으며, AI 결과물은 검토 대상 중 하나로만 사용했다.
